# card-row-interactivity

## Goal
Improve Card Row stencil interactivity - add smooth animations, hover feedback, and transitions similar to what we did for Deep Dive (accordion animation, content stagger, hover shifts, icon morphs, image crossfade, reduced motion coverage).

## Learnings
- Card Row already has 7 transitions (card hover lift, link brand color, track slide, nav hover, dot hover/scale, skeleton pulse, reduced motion block). Not starting from zero.
- Nav buttons currently use JS inline `style.opacity` to hide/show — this bypasses CSS transitions entirely. Needs refactor to CSS class toggle for smooth disabled state.
- The DS button active state convention is `scale(0.97)` for full-size buttons. Nav circle buttons (36px) should use `scale(0.92)` for visible feedback at small size.
- Arrow SVG in card links is 16x16, points 45-degrees up-right. Nudge should be `translate(3px, -3px)` to follow arrow direction, not just `translateX`.
- Deep Dive image crossfade is on `dd__media img` with `opacity var(--duration) var(--ease)` — direct pattern to reuse on `.cr__card-img img`.
- Skeleton pulse uses `calc(var(--duration) * 8)` = 1600ms cycle. Shimmer would be inconsistent with established pulse pattern.
- Dot segments are 32x3 bars, not circles. Width expansion to 48px for active state creates a progress-bar metaphor.
- `overflow-x: clip` on `.cr__row` already clips shadows; adding CSS `mask-image` edge fades risks double-clipping. Lower priority.
- Card entrance stagger requires JS coordination with slide timing — medium complexity. Deferred.
- No lint or typecheck scripts exist in this project. Verification is visual + build only.
